我一直觉得，学习技术最容易出现的问题不是“没有看过”，而是“看过之后没有自己动手实现，也没有留下清晰的记录”。很多知识在刚学的时候似乎理解了，但过一段时间再回头看，只能记得一个模糊的框架，具体的实现细节、设计取舍和问题边界都已经变得不清楚。等到真正需要应用这些知识时，才会发现自己并没有真正掌握它。

**Aqwddda Log** 就是为了解决这个问题而建立的。我希望把推荐系统、RAG和AI工程实践中的学习过程沉淀下来。记录自己学习新知识的背景，动手实现的过程、遇到的具体问题，以及在这个过程中形成的一些思考。

## 为什么选择公开写作

公开写作会迫使我把一个问题讲清楚。相比只在脑子里觉得自己“好像懂了”，写成文章需要重新组织概念、梳理逻辑、补齐细节，并且尽量让别人也能理解。这个过程能够暴露很多知识结构里的空白，避免把“看过”误认为“理解”。

我希望这里的文章不是浮于表面的知识摘抄，而是经过自己思考后的输出。文章应该有清晰的逻辑，能够由浅入深地讲清楚一个问题，也应该尽量包含具体的动手实现，例如可运行的代码、简化示例、公式推导或实验记录。

## 计划记录的方向

这个博客会主要围绕我在学习、工作和研究中遇到的问题展开，包括推荐系统的学习路径、模型理解与工程实现，RAG 和 Agent 系统的学习与实践，以及 AI 项目的代码组织、实验管理和部署经验。

其中会包含代码、公式和实验记录。例如，一个简化的向量相似度可以写为：

$$
\operatorname{sim}(q, d) = \frac{q \cdot d}{\lVert q \rVert \lVert d \rVert}
$$

也会保留足够小、可以直接运行的代码片段：

```python
def reciprocal_rank(relevant_positions: list[int]) -> float:
    if not relevant_positions:
        return 0.0
    return 1.0 / min(relevant_positions)
```

## 长期维护，而不是一次性展示

这个网站选择 Astro、Markdown 和静态部署，是为了让维护成本尽可能低。内容以普通文本文件存在，可以被版本控制，也不依赖数据库或后台系统。这样我可以把更多精力放在内容本身，而不是博客系统的维护上。

我会把这里当作持续更新的工程日志。旧文章可能会因为新的实验结果而修订，但会尽量保留清晰的上下文和边界。比起追求更新频率，我更关心每篇记录能否在未来再次派上用场。

**Aqwddda Log** 的目标很简单：记录真实的学习过程，沉淀可复用的技术理解。
